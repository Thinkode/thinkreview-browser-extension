# ThinkReview - Version 2.2.0 Release Notes

**Release Date:** 19 April 2026

---

## 🎯 What's New

### Implement review findings in your IDE 💻

You can send suggestions, best practices, and security findings straight into **Cursor**, **GitHub Copilot (VS Code)**, or **Claude Code (VS Code)** with one click on each row in the Review tab.

- **Pick your tool:** Use the new control in the extension popup (next to layout) to choose which “Implement” shortcut appears—or hide those buttons if you prefer a read-only panel.
- **Richer prompts:** Actions build a ready-to-paste prompt that includes the MR/PR context so your IDE can match the right local workspace.
- **Same panel:** Works alongside your existing review flow; nothing changes until you use an Implement button.

### Azure DevOps: sign in with your browser ☁️

On **Azure DevOps Services** (dev.azure.com and Visual Studio–hosted orgs), you can rely on **your current browser sign-in** instead of storing a Personal Access Token when that fits how you work.

- **Clearer guidance:** If access is missing, the panel explains whether you need to **sign in / refresh** (cloud session) or **fix your PAT** (on-premises and token-based setups).

---

## 🔧 Additional Improvements

- **Review tab copy behavior:** When you use copy actions on suggestion rows, the experience favors **clean, copy-pasteable code** so you can drop fixes into your editor faster.
- **Polish:** Minor layout and messaging updates in the review panel and popup for IDE assist options.
