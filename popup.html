Azure DevOps Server (Self-Hosted)

On-premises or self-hosted Azure DevOps? Follow these steps to get started:

1
In Azure DevOps, go to User Settings → Personal Access Tokens and create a token with Code (read) scope
2
Paste your PAT in the field below and click Save Token
3
Enter your custom domain (e.g. devops.mycompany.com) and click Add
4
Navigate to any pull request on your instance — ThinkReview activates automatically
Not working? Azure DevOps setup guide

Bitbucket Cloud

Setup required

Bitbucket requires you to grant browser permission and add an app password to enable the extension.

1
Click Allow Bitbucket below to grant browser permission
2
In Bitbucket, go to Profile → App passwords and create one with Pull requests: read scope
3
Enter your Atlassian email and app password below, then save
4
Navigate to any pull request on bitbucket.org

You must grant permission by pressing the button above before using ThinkReview on Bitbucket.
